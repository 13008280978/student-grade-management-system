## 1. 项目概述

### 1.1 功能定位

学生成绩管理系统面向高校教务管理场景，围绕账号、学生档案、课程目录、成绩记录、成绩统计和学业分析进行设计。系统以 JSP + Servlet + JDBC 为核心技术，采用 MVC 分层结构实现页面展示、请求控制、数据模型和数据库访问的职责分离。

系统满足 JSP 程序设计课程对注册、登录、查询、编辑、删除、角色权限、JDBC 数据库操作、页面交互和文档说明的要求，并在基础用户管理功能之上扩展成绩业务功能，形成一个完整可运行的课程作业项目。

主要功能包括：

- 用户认证：注册、验证码校验、登录、注销、SHA-256 密码摘要。
- 账号管理：登录名 / 邮箱 / 日期组合查询、资料编辑、密码重置二次验证、逻辑禁用、物理删除。
- 角色权限：教务管理员和学生用户两类角色，后端统一鉴权。
- 学生档案：学生新增、编辑、查询、归档、删除。
- 课程目录：课程新增、编辑、查询、删除。
- 成绩记录：成绩录入、编辑、查询、删除，限制同一学生同一课程同一学期重复录入。
- 成绩分析：课程平均分、分数段分布、学生均分排名。
- 教务扩展：个人成绩单、绩点换算、学业预警、奖学金候选评定、学分绩点汇总、培养方案达成度。
- 工程增强：HikariCP 数据库连接池、PreparedStatement 防 SQL 注入、中英文界面切换、内嵌 Tomcat 启动。

### 1.2 技术架构图

![学生成绩管理系统技术架构图](docs/diagrams/grade_architecture.png)

系统架构分为浏览器端、Web 表现层、控制层、模型与 DAO 层、数据库层：

- 浏览器端通过 JSP 页面提交表单、查看列表和图表。
- JSP / JSTL / EL 负责页面渲染，JavaScript 负责基础表单校验。
- Servlet Controller 负责接收请求、判断权限、封装参数、调用 Repository。
- Model JavaBean 负责数据封装。
- Repository 通过 JDBC 和 HikariCP 操作 MySQL。
- MySQL 保存账号、角色、学生、课程和成绩等核心数据。

## 2. 需求分析

### 2.1 用例图与用户角色权限

![学生成绩管理系统用例图](docs/diagrams/grade_usecase.png)

系统包含游客、学生用户、教务管理员三类访问身份。游客只能访问注册和登录页面；登录后根据账号角色进入不同功能范围。

| 角色 | 权限说明 |
| --- | --- |
| 游客 | 注册账号、登录系统、切换语言。 |
| 学生用户 learner | 查看首页统计、课程目录、本人学生档案、本人个人成绩单、本人学分绩点汇总和本人学业预警。不能维护账号、学生、课程和成绩基础数据。 |
| 教务管理员 manager | 维护账号、学生档案、课程目录、成绩记录，查看成绩分析、奖学金候选、培养方案达成度等教务分析页面。 |

后端通过 `SessionGate.loginRequired` 判断是否登录，通过 `SessionGate.managerRequired` 判断是否具备教务管理员权限。学生用户即使直接输入管理员接口地址，也会收到 403 权限拒绝响应。

## 3. 数据库设计

### 3.1 ER 图

![学生成绩管理系统 ER 图](docs/diagrams/grade_er.png)

数据库名称为 `grade_center`，包含角色表、账号表、学生档案表、课程表和成绩表。主要关系如下：

- 一个角色可以对应多个账号，账号通过 `role_id` 关联角色。
- 一个学生档案可以绑定一个学生账号，账号删除后学生档案的 `account_id` 自动置空。
- 一个学生可以拥有多条成绩记录。
- 一门课程可以拥有多条成绩记录。
- 成绩表通过唯一约束保证同一学生、同一课程、同一学期只能录入一条成绩。

## 4. 实现说明

### 4.1 MVC 实现细节

| 层次 | 包 / 文件 | 说明 |
| --- | --- | --- |
| View | `src/main/webapp/**/*.jsp` | 页面展示、表单提交、JSTL 条件渲染、EL 表达式取值 |
| Controller | `cn.course.grade.controller` | 处理请求、参数读取、权限校验、调用 Repository、页面跳转 |
| Model | `cn.course.grade.model` | 账号、学生、课程、成绩、统计结果等 JavaBean |
| Repository | `cn.course.grade.dao` | 封装 SQL 查询与更新逻辑 |
| Util | `cn.course.grade.util` | 连接池、密码摘要、权限、限流、绩点工具 |

MVC 请求流程如下：

1. 用户在 JSP 页面填写表单或点击菜单。
2. 请求进入对应 Servlet，例如 `/scores/*` 进入 `ScoreAction`。
3. Controller 校验登录状态和角色权限。
4. Controller 调用 Repository 执行数据库查询或更新。
5. Repository 使用 `PreparedStatement` 参数绑定，避免 SQL 注入。
6. 查询结果封装为 Model 对象。
7. Controller 将数据放入 request，再转发到 JSP 展示；保存或删除成功后重定向到列表页。

### 4.2 核心技术点

| 技术点 | 实现位置 | 说明 |
| --- | --- | --- |
| 密码摘要 | `DigestTool.sha256Hex` | 将密码转为 SHA-256 摘要后保存，不保存明文密码。 |
| 验证码 | `AuthAction.refreshCheckCode` | 生成五位数字验证码并存入 session，登录和注册时校验。 |
| 登录限流 | `ThrottleBox` | 对同一客户端 IP 的失败登录做时间窗口限制，降低暴力破解风险。 |
| 会话鉴权 | `SessionGate` | 统一提供登录校验和管理员权限校验。 |
| JDBC 连接池 | `JdbcKit` | 读取 `jdbc.properties`，使用 HikariCP 创建连接池。 |
| SQL 预处理 | `SqlRows` 和 Repository | 统一使用参数绑定查询，避免拼接用户输入。 |
| 成绩唯一性 | `score_record` + `ScoreRepository.duplicated` | 数据库唯一约束和后端重复检测双重保证。 |
| 绩点换算 | `GpaTool` | 低于 60 分绩点为 0，60 分及以上按 `(分数 - 50) / 10` 计算，最高 5.00。 |
| 图表展示 | JSP + Chart.js | 首页和分析页展示课程均分、分数段分布等图表。 |
| 国际化 | `grade_text_zh_CN.properties` / `grade_text_en_US.properties` | 支持中文和英文界面切换。 |
| 内嵌服务器 | `EmbeddedGradeServer` | 使用内嵌 Tomcat 注册 Servlet，方便通过 Maven 直接运行。 |

### 4.3 关键代码逻辑说明

#### 4.3.1 登录与注册

`AuthAction` 负责 `/signin`、`/signup`、`/signout`、`/language` 四类请求：

- GET `/signin` 和 `/signup`：刷新验证码并进入登录或注册页面。
- POST `/signin`：先检查 IP 是否被限流，再校验验证码，然后通过账号名和密码摘要查询账号。
- POST `/signup`：校验验证码、登录名唯一性、邮箱唯一性，通过后创建学生用户账号。
- GET `/signout`：销毁 session 并返回登录页。
- GET `/language`：切换中英文语言环境。

#### 4.3.2 权限控制

`SessionGate.loginRequired` 用于保护需要登录访问的页面，未登录用户会被重定向到 `/signin`。`SessionGate.managerRequired` 用于保护管理员功能，如果当前账号不是 `manager`，系统返回 HTTP 403，并提示需要教务管理员权限。

#### 4.3.3 成绩录入与重复检测

`ScoreAction.save` 接收学生、课程、学期、分数和备注。保存前调用 `ScoreRepository.duplicated(studentId, courseId, term, id)` 判断是否存在重复成绩。如果重复，则返回成绩表单并显示“同一学生、课程和学期的成绩已存在”。数据库层还设置了 `uk_student_course_term` 唯一约束，确保并发情况下也不能写入重复数据。

#### 4.3.4 成绩分析

`ScoreRepository` 提供以下统计查询：

- `courseAverages`：按课程统计平均分。
- `bands`：按 90-100、80-89、70-79、60-69、0-59 统计分数段。
- `ranking`：按学生平均分生成前 20 名排名。
- `alerts`：筛选低于 70 分的课程，生成学业预警。
- `scholarship`：根据平均分阈值和不及格课程数筛选奖学金候选。
- `creditAudit`：统计通过学分、未通过学分和加权绩点。
- `programProgress`：按班级统计培养方案达成度。

#### 4.3.5 前端表单校验

`assets/js/grade-check.js` 提供注册、成绩、学生表单的前端校验：

- 登录名必须以字母开头，长度 4-16 位。
- 密码长度为 6-18 位，并同时包含字母和数字。
- 邮箱必须符合基本邮箱格式。
- 学号必须为 9 位数字。
- 入学年份限制在 2000 到 2100。
- 学期格式为 `2026-春` 或 `2026-秋`。
- 成绩必须在 0 到 100 之间。

## 5. 运行指南

### 5.1 环境配置步骤

1. 安装 JDK 1.8 或以上版本，并配置 `JAVA_HOME`。
2. 安装 MySQL 5.7 / 8.0。
3. 安装 Maven，并确认命令行可执行 `mvn -v`。
4. 在 MySQL 中执行 `database/grade_center.sql` 初始化数据库。
5. 修改 `src/main/resources/jdbc.properties` 中的数据库连接信息。

`jdbc.properties` 示例：

```properties
jdbc.driverClassName=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/grade_center?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
jdbc.username=root
jdbc.password=你的数据库密码
jdbc.poolSize=10
```

### 5.2 内嵌 Tomcat 运行

在项目目录执行：

```powershell
cd C:\Users\31046\Desktop\jsp\成绩管理系统作业
mvn clean compile exec:java
```

默认访问地址：

```text
http://localhost:8081/grade-center
```

如需修改端口：

```powershell
mvn exec:java -Dapp.port=8091
```

### 5.3 外部 Tomcat 部署

执行 Maven 打包：

```powershell
cd C:\Users\31046\Desktop\jsp\成绩管理系统作业
mvn clean package
```

打包完成后，将 `target/grade-center.war` 复制到 Tomcat 8.5+ 的 `webapps` 目录，启动 Tomcat 后访问：

```text
http://localhost:8080/grade-center
```

### 7.2 课程要求对应关系

| 课程要求 | 项目实现 |
| --- | --- |

| 角色权限 | manager 和 learner 两种角色 |

| 图表可视化 | Chart.js 展示课程均分和分数段分布 |
